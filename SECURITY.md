# Security Policy

## Reporting a vulnerability

If you believe you have found a security issue in this project, please do not open a public issue first.

Instead, report it privately through GitHub security reporting tools if enabled for the repository, or contact the maintainer directly through a private channel.

Please include:

- a description of the issue,
- steps to reproduce it,
- affected versions or commits,
- potential impact.

You can expect an acknowledgement and triage response as soon as reasonably possible.
